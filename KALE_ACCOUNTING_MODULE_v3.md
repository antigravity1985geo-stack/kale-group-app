# KALE GROUP — ACCOUNTING MODULE (v3.0.0)
## მომხმარებლის სახელმძღვანელო & ტექნიკური დოკუმენტაცია

ეს დოკუმენტი წარმოადგენს Kale Group-ის საბუღალტრო და საოპერაციო მოდულის განახლებულ სახელმძღვანელოს. v3.0 ვერსიაში დამატებულია ესტონური მოდელის მოგების გადასახადი, ხელფასების სრული ავტომატიზაცია და დინამიური ფინანსური რეპორტინგი.

---

## 🔝 v3.0-ის მთავარი სიახლეები

1.  **ესტონური მოდელის გადასახადი**: მოგების გადასახადის (15%) ავტომატური გაანგარიშება დივიდენდების გაცემისას.
2.  **ხელფასების ავტომატიზაცია**: ერთი კლიკით ხელფასების დარიცხვა და 20%-იანი საშემოსავლოს ავტო-ფორმირება.
3.  **დინამიური ფინანსური რეპორტები**: P&L, ბალანსი და საცდელი ბალანსი თარიღების ფილტრით.
4.  **CSV/Excel ექსპორტი**: ყველა რეპორტის გადმოწერა ანალიზისთვის.
5.  **RS.ge სინქრონიზაცია**: B2B ზედნადებების და ინვოისების გაუმჯობესებული მართვა.

## 1. ესტონური მოდელი (მოგების გადასახადი)

საქართველოს საგადასახადო კოდექსის (ესტონური მოდელის) მიხედვით, მოგების გადასახადი იბეგრება მხოლოდ მოგების განაწილებისას (დივიდენდი).

### გაანგარიშების ფორმულა:
*   **დივიდენდი (ხელზე)**: `Net Amount`
*   **გადანაწილებული მოგება**: `Net Amount / 0.85`
*   **მოგების გადასახადი (15%)**: `(Net Amount / 0.85) * 0.15`

### საბუღალტრო გატარებები (Journal Entries):
დივიდენდების დეკლარირებისას სისტემა ავტომატურად აკეთებს შემდეგ გატარებას:

| ანგარიში | დასახელება | დებეტი | კრედიტი |
| :--- | :--- | :--- | :--- |
| **5200** | გაუნაწილებელი მოგება | `Amount / 0.85` | |
| **3330** | დივიდენდები გადასახდელი | | `Amount` |
| **3320** | მოგების გადასახადი გადასახდელი | | `(Amount / 0.85) * 0.15` |

## 2. ხელფასების მოდული (HR & Payroll)

ხელფასების მოდულში ინტეგრირებულია საშემოსავლო გადასახადის (20%) ავტომატური დარიცხვა.

### სამუშაო პროცესი:
1.  **HR სექცია**: დაამატეთ თანამშრომელი, მიუთითეთ პოზიცია და "ხელზე ასაღები" ხელფასი.
2.  **Payroll Run**: "Run Payroll" ღილაკზე დაჭერით სისტემა ყველა აქტიურ თანამშრომელზე ქმნის უწყისს.
3.  **ავტო-გატარება**: სისტემა ავტომატურად აგენერირებს ჟურნალის ჩანაწერს:

| ანგარიში | დასახელება | დებეტი | კრედიტი |
| :--- | :--- | :--- | :--- |
| **8100** | ხელფასის ხარჯი (Gross) | `Amount / 0.8` | |
| **3300** | ხელფასი გადასახდელი (Net) | | `Amount` |
| **3310** | საშემოსავლო გადასახდელი | | `(Amount / 0.8) * 0.2` |

## 3. დინამიური ფინანსური რეპორტინგი

v3.0-ში რეპორტები აღარ არის შეზღუდული მხოლოდ კალენდარული თვეებით.

### ხელმისაწვდომი რეპორტები:
*   **მოგება-ზარალი (P&L)**: აჩვენებს შემოსავლებს, COGS-ს და ხარჯებს არჩეულ პერიოდში.
*   **ბალანსი (Balance Sheet)**: აჩვენებს კომპანიის ფინანსურ მდგომარეობას კონკრეტული თარიღისთვის.
*   **საცდელი ბალანსი (Trial Balance)**: აჩვენებს ნაშთებს (საწყისი, ბრუნვა, საბოლოო) ყველა ანგარიშზე.

### ექსპორტი:
ყველა რეპორტს აქვს **Export CSV** ფუნქცია, რომელიც ავტომატურად აგენერირებს Excel-თან თავსებად ფაილს.

## 4. RS.ge & Invoicing

ინვოისების მოდული სრულად სინქრონიზებულია RS.ge-სთან.

*   **B2C**: ონლაინ გაყიდვები (BOG/TBC) ავტომატურად ქმნის ინვოისს სერიით `INV-`.
*   **B2B**: ორგანიზაციული გაყიდვებისას შეგიძლიათ შეავსოთ სრული ფორმა (დასახელება, ს/კ, მისამართი) და ერთი კლიკით ატვირთოთ RS.ge-ზე.
*   **ზედნადებები (Waybills)**: სისტემა მხარს უჭერს საავტომობილო ტრანსპორტირების ზედნადებების გაწერას და დასრულებას.

---

## 5. ზედნადებები და მარაგების მართვა (Waybills & Inventory)

სისტემა ავტომატურად მართავს მარაგების თვითღირებულებას FIFO (First-In, First-Out) მეთოდით.

### RS.ge Incoming Waybills:
*   **ავტო-იმპორტი**: სისტემა კითხულობს მომწოდებლის მიერ გამოწერილ ზედნადებებს.
*   **Mapping**: შემოსული საქონელი ავტომატურად უკავშირდება ინვენტარის პროდუქტებს.
*   **FIFO ფენები**: ყოველი ჩაბარება (GRN) ქმნის ახალ "თვითღირებულების ფენას" (Cost Layer), რაც უზრუნველყოფს მოგების ზუსტ გაანგარიშებას გაყიდვისას.

### ტრანსპორტირების ხარჯი (Account 1330):
თუ შესყიდვას ახლავს ტრანსპორტირების ხარჯი, სისტემა საშუალებას გაძლევთ ეს ხარჯი პირდაპირ ასახოთ საქონლის თვითღირებულებაში, რაც ზრდის აქტივის ღირებულებას და აკორექტირებს მომავალ COGS-ს.

## 6. API Endpoints (Technical)

დეველოპერებისთვის დამატებული ახალი ენდპოინტები:

```bash
POST /api/accounting/payroll/run

# რეპორტები თარიღების მიხედვით
GET /api/accounting/reports/profit-loss?startDate=2026-01-01&endDate=2026-03-31
GET /api/accounting/reports/balance-sheet?date=2026-04-09
GET /api/accounting/reports/trial-balance?startDate=2026-01-01&endDate=2026-04-09
```
